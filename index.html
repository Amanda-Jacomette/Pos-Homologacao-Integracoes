01
Pós-Homologação

Guia de Ativação
AutoTEF

Oriente o parceiro na ativação da solução AutoTEF, garantindo que as permissões de Microtef estejam corretamente configuradas.

Pré-requisitos de Ativação
🏷️
Stone Code Ativo
O cliente deve possuir um Stone Code ativo e válido antes de iniciar qualquer processo de ativação do AutoTEF.
🔑
SAK de Microtef (MicroPOS)
Essencial para o funcionamento. Sem a SAK de Microtef habilitada, a ativação falhará. Verifique este item com atenção.
⚠️
Atenção: A ausência da SAK de Microtef é a causa mais comum de falha na ativação. Valide este pré-requisito antes de prosseguir.
Fluxos de Disponibilização
✅
Novo Credenciamento Recomendado
O parceiro ou time de Inbound credencia o cliente diretamente com o perfil AutoTEF (com micropos). O Stone Code já nasce com as chaves corretas.
🔄
Migração TEF → AutoTEF
Para clientes que já usam TEF e desejam manter as taxas atuais. A habilitação da SAK de Microtef e a solicitação de migração devem ser feitas pelo parceiro junto ao time souparceiro@stone.com.br.
Passo a Passo Técnico
1
Envio do Stone Code

O cliente envia o Stone Code para o parceiro responsável pela ativação.

2
Instalação do AutoTEF Slim

Siga o guia oficial de instalação: autotef.readme.io → Instalando o Slim.

3
Configurações Iniciais — Ambiente de Produção

Nas configurações iniciais, certifique-se de setar o ambiente como produção. Usar ambiente de homologação é causa frequente de falha silenciosa.

4
Troubleshooting — Erro na Ativação

Se a ativação falhar mesmo com o Stone Code correto, o parceiro deve acionar o time souparceiro@stone.com.br para verificar a habilitação da SAK de Microtef ou solicitar novo credenciamento com o perfil AutoTEF correto.

Suporte Escalado via Guru

🎫 Abertura de Ticket via Guru — Partner Hub

Se o problema persistir mesmo com o Stone Code correto, abra um ticket via Guru no Partner Hub direcionado ao time de Integrações. É obrigatório anexar os logs de ativação — sem eles o chamado não será processado. Inclua também o Stone Code do cliente e o histórico de tentativas.

02
Pós-Homologação

Guia de Habilitação
Connect Stone

Instrua o parceiro sobre como habilitar a solução Connect Stone para qualquer cliente via Partner Hub — sem necessidade de chamado ou aprovação manual.

Métodos de Liberação (Escolha um)
🛒
Via Funil de Vendas
→
Realizado durante o credenciamento de um novo cliente pelo parceiro ou pelo time de Inbound.
→
No próprio ato do credenciamento é possível ativar o Connect Stone e, ao finalizar, já coletar a chave secreta sk_....
→
Ideal para garantir que o cliente entre na base já com o produto ativo.
⚙️
Via Ativação de Produto
1
Acesse o Partner Hub com suas credenciais.
2
Navegue até Integrações → Ativação de Produto.
3
Localize e selecione a solução Connect Stone homologada.
4
Clique em "Adicionar Lojista" e insira os dados do cliente.
5
Colete a chave secreta sk_... gerada — ela é necessária para a vinculação no sistema do parceiro.
💡
Abrangência: A liberação pode ser feita para qualquer cliente, não apenas os da carteira do parceiro. A habilitação é autônoma e imediata.
FAQ Rápido
Após a homologação, quando o produto fica disponível para liberação no Hub? Prazo
O produto pode levar até 24 horas para aparecer disponível para liberação no Partner Hub após a conclusão da homologação. Se não aparecer após esse prazo, verifique se a homologação foi finalizada e aprovada antes de abrir chamado.
Erro na ativação no portal — o que verificar? Erro
Verifique se o cliente já está ativo em outro parceiro. Um mesmo lojista não pode estar vinculado a dois parceiros simultaneamente para o mesmo produto. É necessário desativar no parceiro anterior antes de ativar no atual.
Onde encontrar documentação e suporte para dúvidas? Suporte
Consulte a seção de documentação e treinamentos disponível dentro do próprio Partner Hub. É a fonte mais atualizada e oficial para parceiros.
Suporte Escalado via Guru

🎫 Abertura de Ticket via Guru — Partner Hub

Se o problema persistir mesmo com o Stone Code correto, abra um ticket via Guru no Partner Hub direcionado ao time de Integrações. É obrigatório anexar todas as evidencias possíveis do erro em questão — sem eles o chamado não será processado. Inclua também o Stone Code do cliente e o histórico de tentativas.

03
Pós-Homologação

Guia de Distribuição
SDK Android

Detalha as formas de disponibilizar um aplicativo Android homologado para os terminais Smart da Stone.

Ciclo de Vida do App
🧪
Fase Piloto
Após a homologação, o app entra automaticamente em fase de Piloto. Nesta etapa, o app é validado pelo parceiro em ambiente controlado (por Stone Code) antes da liberação geral.
🚀
Fase Rollout
Após validação do Piloto, o app é liberado para Rollout — a disponibilização geral para toda a base de lojistas elegíveis.
Métodos de Liberação (Escolha um)
🛒
Via Funil de Vendas
→
Disponível durante o credenciamento de um novo cliente ou para clientes já credenciados que ainda não tiveram o app ativado.
→
O parceiro acessa o funil do cliente no Partner Hub e ativa o produto diretamente por lá.
→
Ideal para ativação pontual por cliente.
⚙️
Via Ativação de Produto
1
Acesse Integrações → Ativação de Produto no Partner Hub.
2
Selecione o App homologado na lista disponível.
3
Clique em Adicionar Lojistas.
4
Insira o Stone Code e o CNPJ do cliente e confirme.
Instalação no Terminal (Maquininha)
📲
Instalação Automática
Após a liberação, o app é instalado automaticamente na maquininha. O terminal precisa estar conectado ao Wi-Fi para receber a atualização.
⏱️
Tempo de Sincronização
Após a liberação no Hub, o app aparece na máquina em até 3 horas. Caso precise da instalação de imediato, reinicie o terminal.
📶
Atenção: O terminal deve estar obrigatoriamente conectado ao Wi-Fi para receber a instalação. Se o app não aparecer no prazo esperado, confirme a conexão e reinicie o terminal.
Verificação de Hardware

🖥️ Compatibilidade do Terminal

Certifique-se de que o modelo da maquininha do cliente é compatível com os modelos selecionados durante o processo de homologação. Incompatibilidades de hardware podem causar falhas na instalação ou no funcionamento do app mesmo após a liberação correta no Hub.

Suporte Escalado via Guru

🎫 Abertura de Ticket via Guru — Partner Hub

Se houver algum problema no processo, abra um ticket via Guru no Partner Hub direcionado ao time de Integrações. É obrigatório anexar todas as evidencias possiveis do erro em questão — sem elas o chamado não será processado. Inclua também o Stone Code do cliente e o histórico de tentativas.
